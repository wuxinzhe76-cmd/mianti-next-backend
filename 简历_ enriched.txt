基于 Spring Boot、MyBatis-Plus 实现的面试刷题平台（含 AI 面试与在线判题）    2025.6 - 2025.8

• 责任描述：参与设计与开发一个高可用的题库管理系统，采用前后端分离架构，实现了用户认证、题库管理、题目管理、AI 模拟面试、在线代码判题等核心功能模块。

• 项目简介：负责系统整体架构设计，遵循 RESTful 风格设计 API 接口，基于 Spring Boot + MyBatis-Plus 构建后端服务。负责用户模块开发，独立完成用户注册/登录、微信授权登录功能，基于 Sa-Token 实现 RBAC 权限认证。在题目搜索场景中，基于 Elasticsearch 实现全文检索，并设计数据库降级方案；利用 Sentinel 实现接口限流（QPS 控制）与熔断降级，防止爬虫恶意访问；基于 Redis + Redisson 实现热点数据缓存，通过京东 HotKey 探测实现 Caffeine 本地缓存加速；基于 AOP 实现统一权限校验与操作日志拦截；利用 Redis 计数器实现访问频率监控，设计实现爬虫检测与自动封号机制；基于 Druid 实现数据库连接池监控与慢 SQL 日志记录。

• AI 模拟面试：基于 Spring AI Alibaba + 通义千问（DashScope）构建 AI 面试官，采用结构化输出（.entity() 强类型反序列化）强制 LLM 返回严格 JSON，通过 actionDirective（DEEP_DIVE / NEXT_QUESTION / END_INTERVIEW）实现 AI 驱动的面试状态机，后端根据指令自动完成话题深入、题目切换、会话终结等路由分发。设计双模式面试（指定题库针对性练习 / 随机大厂全真模拟），通过 System Prompt 注入面试官人设与行为规则。实现 Redis 热数据存储方案（会话期间全部数据驻留 Redis，TTL 自动过期），结合滑动窗口截取最近 N 轮对话历史控制 Token 消耗。面试结束后将 Redis 数据批量刷入 MySQL 并通过 RabbitMQ 异步生成面试报告。

• 在线代码判题系统（Online Judge）：设计并实现 ACM 风格的代码判题引擎，支持 Java / C++ / Python / JavaScript 多语言编译执行。采用策略模式抽象 CodeSandbox 接口，通过 ProcessBuilder + Runtime.exec() 实现代码沙箱本地执行，支持编译型语言（Java/C++）先编译后执行、解释型语言（Python/JS）直接运行。判题过程采用 CompletableFuture 异步处理，提交后即刻返回 Submission ID，后台逐条跑测试用例并 fail-fast 终止。实现超时控制（Process.waitFor + destroyForcibly）、内存限制（JVM -Xmx 参数注入）、输出规范化（跨平台换行符统一）， verdict 覆盖 AC / WA / TLE / MLE / RE / CE 八种状态。前端集成 Monaco Editor（VS Code 内核）实现语法高亮代码编辑，配合 2s 轮询实现判题结果实时展示。

• 题目统计与通过率计算：代码判题 ACCEPTED 后自动更新题目提交数、通过数、通过率等统计指标，支持部分计分（OI 赛制）——每条测试用例独立赋分，总分累加。
